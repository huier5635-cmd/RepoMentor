# RepoMentor 3–5 分钟 Demo 录屏脚本

## 录屏目标

展示 RepoMentor 满足任务书中的三项交付要求：

- 可运行 Demo
- 系统展示
- 完整交互轨迹

建议录屏时长：3–5 分钟。建议使用本地地址：

- 后端：`http://127.0.0.1:8000`
- 前端：`http://127.0.0.1:5173`

## 0. 开场，约 20 秒

讲解词：

> 这是 RepoMentor，一个面向新贡献者的多智能体仓库学习助手。它的目标不是替代开发者读代码，而是把仓库中的代码、文档、测试、运行命令、开发流程和 Issue 解析成可验证的学习路径，并在问答时给出证据来源。

画面：

- 打开 `http://127.0.0.1:5173`
- 展示 RepoMentor 首页和仓库输入框

## 1. 输入 GitHub 仓库链接，约 25 秒

操作：

1. 在输入框中输入一个公开 GitHub 仓库链接。
2. 点击“分析”。
3. 等待进入 `/repos/{repo_id}` 用户页。

讲解词：

> 这里输入 GitHub 仓库链接后，后端会抓取仓库结构，并由多个 Worker 分别解析文件、符号、依赖、文档、测试关系、Issue、开发流程和代码解释。模型不会决定仓库事实，事实先进入 Repository Intelligence Graph。

## 2. 展示仓库概览，约 25 秒

操作：

- 停留在“仓库概览”。
- 指出文件数、语言、核心目录、入口文件、安装/启动/测试命令。

讲解词：

> 仓库概览回答的是新贡献者的第一组问题：这个仓库是什么、从哪里开始看、怎么安装、怎么运行、怎么测试。入口文件和命令来自 README、docs、配置文件和脚本解析，不是模型猜出来的。

## 3. 展示代码地图，约 30 秒

操作：

- 点击“代码地图”。
- 展示文件类型、核心模块、测试关系和文档关系。

讲解词：

> 代码地图把文件分成文档、源码、测试、配置、数据、前端页面、样式、许可证和参考文献等更友好的类型。对学习者来说，重要的不是看到所有内部 JSON，而是知道哪些文件应该先读、哪些测试保护了哪些模块。

## 4. 展示学习路径，约 35 秒

操作：

- 点击“学习路径”。
- 展示 Learning Path V2 的步骤。
- 可切换“架构导览”“模块卡”“双语术语”。

讲解词：

> 学习路径把仓库拆成项目目标、启动方式、主流程、核心模块、测试边界、开发流程、入门任务和改动演练。每一步都有具体文件、命令、预期观察和常见误区。双语术语表会保留命令、路径、URL 和代码符号，避免翻译破坏可执行信息。

## 5. 展示开发流程，约 30 秒

操作：

- 点击“开发流程”。
- 展示安装命令、启动命令、测试命令、PR/CI/风险提示。

讲解词：

> Development Workflow Worker 会从 README、docs、配置文件、CI 和 PR 模板中恢复贡献流程。它会区分 setup rule、test rule、code style、PR rule、CI rule 和项目安全策略；缺失的信息会标记为风险或不确定项，而不是编造。

## 6. 展示推荐任务，约 30 秒

操作：

- 点击“推荐任务”。
- 如果 open issue 为 0，展示 internal first tasks。

讲解词：

> 如果 GitHub open issue 为 0，RepoMentor 不会只说“没有 issue”。它会基于仓库结构生成 internal first tasks，例如跑通测试、阅读架构文档、运行 demo、检查 README quick start。这些任务会明确标记“不是 GitHub Issue”，source 是 internal_suggestion。

## 7. 证据问答，约 45 秒

操作：

1. 点击“QA”。
2. 输入问题：`这个项目怎么启动`
3. 提交问题。
4. 展示回答、可运行命令、风险提示、证据来源。

讲解词：

> 这里问“这个项目怎么启动”。RepoMentor 会优先返回用户能执行的答案，例如先安装依赖，再运行 demo 脚本，再运行测试。回答下方会显示证据来源，比如 README、RepositoryGraph、命令解析结果和相关文件。SelfCheck 会检查结论是否有证据支持，命令是否有来源。

## 8. 切换调试控制台，约 50 秒

操作：

1. 点击“调试控制台”。
2. 展示 Agent Flow。
3. 展示 Data Layer。
4. 展示 Retrieval Trace / Shared Memory。
5. 展示 SelfCheck。
6. 展示 FinalAnswer JSON。

讲解词：

> 用户页隐藏内部实现信息，调试控制台则用于展示完整交互轨迹。这里可以看到 Agent Flow、8 个 Worker 的输出、Data Layer 图谱、共享记忆、检索证据、Evaluator/Optimizer 和最终 FinalAnswer JSON。这样既保证用户体验简洁，也满足系统展示和可调试性的要求。

## 9. 收尾，约 20 秒

讲解词：

> 目前 RepoMentor 的基础链路已经跑通：仓库解析、代码地图、学习路径、开发流程、推荐任务、证据问答和调试轨迹都可以运行。压力测试中的早期失败主要来自 evidence 覆盖不足，不是系统崩溃；后续已经通过 EvidenceBuilder、文档/测试映射、命令去重和 SelfCheck 加强。下一阶段会继续完善 EvidenceBuilder、DeepSeek 实测、LangGraph 默认编排和部署展示。
